# Model Catalog

TT-Studio's model catalog is automatically synced from [TT Inference Server](https://github.com/tenstorrent/tt-inference-server). The authoritative list of all supported models lives there — the catalog in this repo is a local snapshot used at deployment time.

## How the Sync Works

Model metadata (supported hardware, inference engine, status) is maintained in `app/backend/shared_config/models_from_inference_server.json`. This file is generated by pulling the latest model list from the TT Inference Server release.

To sync the catalog with the latest TT Inference Server release:

```bash
python3 app/backend/shared_config/sync_models_from_inference_server.py
```

The sync script fetches the current model definitions and regenerates the JSON. TT-Studio uses this file at startup to populate the model catalog and filter it by the detected hardware (via TT-SMI).

**You do not need to sync manually** — TT-Studio ships with a catalog snapshot that matches its tested TT Inference Server version. Run the sync only when upgrading to a newer TT Inference Server release.

## Status Definitions

| Status | Description |
|--------|-------------|
| **COMPLETE** | Production-validated; suitable for sustained use |
| **FUNCTIONAL** | Works correctly; suitable for evaluation and development |
| **EXPERIMENTAL** | Early access; behavior may vary |
